Pebble × GitHub

GitHub Watcher — Setup

A Pebble watchapp for at-a-glance CI status, a QR bridge to jump to any PR on a bigger screen, and one-tap actions like re-running a failed job. Setup takes about 3 minutes.

Before you start

  • A Pebble watch (or the Pebble emulator via the SDK / CloudPebble).
  • A GitHub account with access to the repositories you want to watch.
How sign-in works. The app uses GitHub's OAuth device flow — you approve it on github.com and pick which repositories it may see. No password or token is ever typed on the watch, access is limited to the repos you choose, and you can revoke it anytime from your GitHub settings.

Set up the app

1

Install the watchapp

Install GitHub Watcher from the Rebble appstore, or build it yourself:

pebble build
pebble install --emulator basalt   # or --phone <ip> for a real watch

You can also import the repo into CloudPebble and build there.

2

Sign in on the watch

On first launch the watch shows a short code. On your phone or computer, go to github.com/login/device, enter the code, and approve. GitHub will ask which repositories to grant access to — pick the ones you want to watch. The watch finishes signing in on its own; no typing on the watch.

3

Choose what to watch

Open the app's settings in the Pebble phone app to add repositories / branches / pipelines, set the poll interval, pick the glance target, and toggle timeline pins (on by default). Prefer a token instead of signing in? Paste a fine-grained personal access token here — the field lists the exact least-privilege permissions it needs.

Good to know

  • Rate limits: the app polls on a schedule and uses conditional (ETag) requests, staying well within GitHub's authenticated 5,000 requests/hour (counted per user).
  • "No access" for a repo? Add it to the app's access: GitHub → Applications → GitHub Watcher → Configure, then select the repo.
  • Token storage: the sign-in token lives in the phone app's storage (protected by the OS app sandbox, not encrypted at rest). It never leaves the phone for the watch, is least-privilege, and is revocable anytime.
  • Timeline pins use the Rebble-hosted timeline service. If it's unavailable, build-done alerts still arrive locally on the watch.

For maintainers & self-hosting

Skip this unless you're forking or self-hosting your own build. The published app already includes a GitHub App client ID; end users never register anything. To run your own copy, register one GitHub App (once) and drop its public client ID into the build.

A

Register a GitHub App

Open github.com/settings/apps/new:

GitHub App nameanything unique, e.g. GitHub Watcher (yourname)
Homepage URLyour repo or project URL
Callback URLleave blank — device flow never redirects
  • ✓Enable Device Flow — required for sign-in.
  • ✗Webhook → Active — uncheck; no webhook needed.
  • •Where can this be installed? Choose Any account if you'll distribute it to others; Only on this account if it's just for you.

Grant only these Repository permissions (everything else: No access):

PermissionLevelUsed for
MetadataReadrequired baseline
ActionsRead & writerun status + re-run failed jobs
ChecksReadbuild / check status
Commit statusesReadstatus API
Pull requestsRead & writeview + merge when green
ContentsReadresolve branch / commit
Distributing to others? After creating the app, on the Advanced / public settings make it public so anyone can install it. Verification (the blue badge) is optional — unverified apps work; the consent screen just notes they're unverified.
B

Ship the Client ID

Copy the app's Client ID (e.g. Iv23li…, public and safe to commit) into src/pkjs/config.js:

// src/pkjs/config.js
module.exports = {
  GITHUB_CLIENT_ID: "Iv23liXXXXXXXXXXXXXX",
};

No client secret is needed — device flow is designed for clients that can't keep secrets. Rebuild, and every install of your app signs in against it.
